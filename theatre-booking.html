Theatre Booking

A vast collaborative full-stack e-commerce system handling complex seating matrices, concurrent transactional flows, and tiered subscription models. Engineered with a robust Selenium testing pipeline and custom Webpack bundling.

The Challenge

Overview & Scope

Moving beyond a simple reservation prototype, this project evolved into a comprehensive event management and e-commerce platform. Handling concurrent ticket reservations in a live environment requires tight database rules, stateful client routing, and an impregnable checkout flow.

Architected as a highly optimized Multi-Page Application (MPA), the system isolates execution contexts across pages using advanced Webpack bundling. We established an end-to-end booking UX spanning multiple state-dependent pages (Seat Selection → Discounts → Summary → Checkout), supported by secure Firebase authentication and rigorous automated testing.

My Role & Contributions

As a core engineer on this 4-person team, I took ownership of the testing infrastructure and the complex e-commerce state management:

  • Automated Testing Pipeline: Configured and executed the k6 load-testing suite to simulate concurrent traffic, and implemented Jest unit-testing for the checkout pricing logic.
  • Dynamic Pricing Engine: Built the algorithmic backend for ticket pricing, linking it securely to Firebase Auth to apply tiered Loyalty Discounts dynamically based on active user sessions.
  • UI & Asset Management: Engineered the promotional DOM slideshows and resolved complex CSS layout shifts within the Webpack asset pipeline to ensure pixel-perfect rendering across varying viewports.

Core Technology Stack

  • Firebase Backend (BaaS): Auth (`connectAuthEmulator`), Firestore CRUD, real-time listeners.
  • Webpack Bundler Engine: Minifies ES6+ JS, transpiles SCSS, custom asset chunking.
  • jQuery UI Control: Heavy dynamic DOM manipulation for physical theatre seating grids.
  • Test Driven Dev: Firebase Emulators isolate tests from production datasets.

Architecture

E-Commerce & Data Ops

Stateful E-Commerce Flow

  • Sequential Checkout: Users traverse a strict path from Event Selection to Seat Maps, Category Discounts, and final Payment.
  • Subscription Engine: Tiered membership models (`SubscriptionCheckout`) granting specific account privileges and discounts.
  • Stateless Session Cart: Because MPAs destroy true JS state on navigation, we leverage the Web Storage API (localStorage) to serialize, cryptographically sign, and safely pass cart data between independent pages.

Database & Admin Ops

  • Concurrency Control: Firebase Firestore documents securely track seat IDs to prevent race-condition double bookings.
  • Admin Event Creation: Built-in management portals (`CreateEvent`) for administrators to dynamically deploy new shows into the database.
  • Relational Mapping: NoSQL associations linking user UUIDs to booking histories, active subscriptions, and payment receipts.

Deep Dive

Advanced Engineering

To deliver a truly robust system, the underlying codebase tackles non-trivial computer science and application state challenges.

Procedural UI Matrices

The physical theatre layout is asymmetrical. Instead of hardcoding HTML, the DOM is procedurally generated using multi-dimensional array algorithms (e.g., seatConfigStall). The script calculates dynamic rows and intersections, binding unique data-seatValue IDs and instantly cross-referencing them against Firebase collision maps to lock occupied nodes before rendering.

MPA Asset Pipeline

A custom webpack.config.js iterates through 13 unique Application Entry Points. Using HtmlWebpackPlugin and MiniCssExtractPlugin, it injects precisely scoped JavaScript and compiled CSS into individual HTML templates. This ensures a user logging in never downloads the exhaustive 23kb script required for the Booking Form, drastically minimizing Time-To-Interactive (TTI).

Assurance

Quality Assurance Pipeline

Given the complexity of simulated financial transactions, membership discounts, and seating state-management, we implemented a rigorous set of test boundaries.

Testing Methodology Technical Implementation
End-to-End Blackbox Testing Selenium WebDriver and ChromeDriver execute programmatic UI workflows, driving an automated Chrome instance through the entire funnel (Seat selection → Discount → Summary verification).
Unit & Integration Testing Jest and Babel validate the algorithmic outcomes of the booking summaries, pricing calculators, and component interactions.
Load Stress Testing k6 deployed to bombard the application infrastructure with concurrent simulated users to test Firebase response latency and connection limits.
